<objective>
## Goal
Convert ACTIVE.md and BACKLOG.md into structured JSON data surfaces with injection hooks and MCP tool access. This is the payoff phase — the infrastructure from Phases 1-3 becomes user-facing here.

## Purpose
ACTIVE.md and BACKLOG.md are currently expensive @-mentioned markdown files that Claude must parse every time. Converting them to structured JSON surfaces means:
- Hooks inject compact summaries every prompt (cheap, passive awareness)
- MCP tools allow structured CRUD (add items, update status, archive)
- Claude doesn't need to parse free-form markdown anymore
- Data is searchable, queryable, and extensible

## Output
- `.base/data/active.json` — Structured active work items migrated from ACTIVE.md
- `.base/data/backlog.json` — Structured backlog items migrated from BACKLOG.md
- `.base/hooks/active-hook.py` — Injection hook for active work awareness
- `.base/hooks/backlog-hook.py` — Injection hook for backlog awareness
- Both surfaces registered in workspace.json
- Both hooks registered in .claude/settings.json
- Source package mirrors for both hooks
</objective>
